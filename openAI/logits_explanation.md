# Logits 详解：理解模型输出的原始分数

## 什么是 Logits？

**Logits** 是模型最后一层（通常是全连接层）输出的**原始分数**，还没有经过 softmax 归一化。

```python
inputs = torch.tensor([
    [2.0, 1.0, 0.5, 0.1],  # 样本 0 的 logits
    [0.5, 2.5, 1.0, 0.3],  # 样本 1 的 logits
    [1.0, 0.5, 2.0, 0.8]   # 样本 2 的 logits
])  # shape: [3, 4]
```

## 数值含义

### 1. 基本结构

- **形状**: `[batch_size, num_classes]` = `[3, 4]`
  - 3 个样本（batch_size = 3）
  - 4 个类别（num_classes = 4）

- **每个值**: 表示模型对该类别的"原始置信度分数"
  - **数值越大** → 模型认为该样本更可能属于这个类别
  - **数值越小** → 模型认为该样本不太可能属于这个类别
  - **可以是负数、零或正数**（没有范围限制）

### 2. 逐样本分析

#### 样本 0: `[2.0, 1.0, 0.5, 0.1]`

```
类别 0: 2.0  ← 最高分，模型最倾向于这个类别
类别 1: 1.0
类别 2: 0.5
类别 3: 0.1  ← 最低分，模型最不倾向于这个类别
```

**含义**: 模型认为样本 0 最可能属于类别 0

#### 样本 1: `[0.5, 2.5, 1.0, 0.3]`

```
类别 0: 0.5
类别 1: 2.5  ← 最高分，模型最倾向于这个类别
类别 2: 1.0
类别 3: 0.3
```

**含义**: 模型认为样本 1 最可能属于类别 1

#### 样本 2: `[1.0, 0.5, 2.0, 0.8]`

```
类别 0: 1.0
类别 1: 0.5
类别 2: 2.0  ← 最高分，模型最倾向于这个类别
类别 3: 0.8
```

**含义**: 模型认为样本 2 最可能属于类别 2

## 从 Logits 到概率

### Softmax 转换

Logits 经过 softmax 后变成概率分布：

```python
import torch
import torch.nn.functional as F

inputs = torch.tensor([
    [2.0, 1.0, 0.5, 0.1],
    [0.5, 2.5, 1.0, 0.3],
    [1.0, 0.5, 2.0, 0.8]
])

probs = F.softmax(inputs, dim=1)
```

**样本 0 的计算过程**:
```
logits = [2.0, 1.0, 0.5, 0.1]

1. 计算 exp(x):
   exp(2.0) = 7.389
   exp(1.0) = 2.718
   exp(0.5) = 1.649
   exp(0.1) = 1.105

2. 计算总和:
   sum = 7.389 + 2.718 + 1.649 + 1.105 = 12.861

3. 计算概率:
   p(类别0) = 7.389 / 12.861 = 0.575 (57.5%)
   p(类别1) = 2.718 / 12.861 = 0.211 (21.1%)
   p(类别2) = 1.649 / 12.861 = 0.128 (12.8%)
   p(类别3) = 1.105 / 12.861 = 0.086 (8.6%)

结果: [0.575, 0.211, 0.128, 0.086]
```

**所有样本的概率**:
```
probs = [
    [0.575, 0.211, 0.128, 0.086],  # 样本 0: 类别 0 概率最高
    [0.123, 0.676, 0.165, 0.036],  # 样本 1: 类别 1 概率最高
    [0.169, 0.062, 0.576, 0.193]   # 样本 2: 类别 2 概率最高
]
```

## Logits 的来源

### 在神经网络中的位置

```
输入数据 → [隐藏层1] → [隐藏层2] → ... → [全连接层] → Logits → Softmax → 概率
                                                              ↑
                                                        这里就是 inputs
```

### 实际例子：图像分类

```python
# 假设是 CIFAR-10 图像分类（10 个类别）
# 输入: 一张猫的图片 (32x32x3)
# 经过 CNN 特征提取后，得到特征向量
# 经过全连接层，得到 logits:

logits = [2.5, -1.2, 0.8, 3.1, -0.5, 1.2, 0.3, -2.1, 1.5, 0.1]
#         飞机  汽车   鸟   猫   鹿   狗   青蛙  马   船   卡车
#         ↑
#      类别 3 (猫) 的分数最高 (3.1)

# 经过 softmax:
probs = [0.05, 0.01, 0.03, 0.78, 0.02, 0.04, 0.02, 0.00, 0.04, 0.01]
#        飞机  汽车   鸟   猫   鹿   狗   青蛙  马   船   卡车
#                                    ↑
#                              78% 概率是猫
```

## Logits 的数值范围

### 特点

1. **无范围限制**: 可以是任意实数（负数、零、正数）
   ```python
   logits = [-5.2, 3.1, 0.0, -1.5]  # 完全合法
   ```

2. **相对大小重要**: 重要的是 logits 之间的**相对差值**，而不是绝对值
   ```python
   # 这两组 logits 经过 softmax 后概率相同:
   logits1 = [2.0, 1.0, 0.0]
   logits2 = [12.0, 11.0, 10.0]  # 都加了 10
   # 因为 softmax 只关心相对大小
   ```

3. **差值越大，置信度越高**: 
   ```python
   # 高置信度预测
   logits = [5.0, 1.0, 0.5]  # 类别 0 明显最高
   probs ≈ [0.98, 0.02, 0.01]  # 非常确定
   
   # 低置信度预测
   logits = [2.1, 2.0, 1.9]  # 三个类别分数接近
   probs ≈ [0.35, 0.33, 0.32]  # 不确定
   ```

## 在损失计算中的作用

### 交叉熵损失使用 Logits

```python
# 直接使用 logits（推荐，数值稳定）
loss = F.cross_entropy(inputs, targets)
# inputs 是 logits，不需要先计算 softmax

# 等价于（但不推荐，数值不稳定）
probs = F.softmax(inputs, dim=1)
loss = -torch.log(probs.gather(1, targets.unsqueeze(1)))
```

**为什么直接用 logits？**
- `F.cross_entropy` 内部使用 `log_softmax`，数值更稳定
- 避免先计算 softmax 再取 log 的数值精度问题

## 总结

| 特性 | 说明 |
|------|------|
| **定义** | 模型最后一层输出的原始分数 |
| **范围** | 任意实数（无限制） |
| **含义** | 数值越大，模型越倾向于该类别 |
| **转换** | 通过 softmax 转换为概率 |
| **用途** | 直接用于计算交叉熵损失（数值稳定） |
| **形状** | `[batch_size, num_classes]` |

**关键理解**:
- Logits 是"未归一化的置信度分数"
- 通过 softmax 变成概率分布
- 在损失计算中，直接使用 logits 更稳定高效
