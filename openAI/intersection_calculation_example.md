## 输入数据准备

### 步骤 1: 模型预测概率 (inputs)

```python
import torch
import torch.nn.functional as F

# inputs: 经过 softmax 的概率, shape (2, 4, 4)
inputs = torch.tensor([

        [0.122, 0.248, 0.236, 0.071],   # 类别 2 在 4 个位置的概率
        [0.110, 0.225, 0.289, 0.780]    # 类别 3 在 4 个位置的概率
    ],

        [0.045, 0.300, 0.250, 0.200],   # 类别 2

```

### 步骤 2: 真实标签的 One-Hot 编码 (targets_one_hot)

```python
# targets: 真实类别索引, shape (2, 4)
targets = torch.tensor([
    [1, 0, 0, 3],  # 样本 0: [B-PER, O, O, B-LOC]
    [0, 0, 0, 0]   # 样本 1: [O, O, O, O]

# 转换为 one-hot 编码
num_classes = 4
targets_one_hot = F.one_hot(targets, num_classes).permute(0, 2, 1).float()

print(f"targets_one_hot shape: {targets_one_hot.shape}")  # torch.Size([2, 4, 4])
```

```
  类别 2: [0.0, 0.0, 0.0, 0.0]  # 没有位置是类别 2

  类别 0: [1.0, 1.0, 1.0, 1.0]  # 所有位置都是类别 0

  类别 2: [0.0, 0.0, 0.0, 0.0]
  类别 3: [0.0, 0.0, 0.0, 0.0]
```

## 步骤 3: 计算交集

## 可视化理解

### 交集的物理含义

**交集**衡量的是：**在所有样本和所有位置上，模型预测概率和真实标签的重叠程度**。

```
对于类别 0:
  样本 0: 位置 1,2 是类别 0 (真实)
          → 模型预测概率: 0.225 + 0.261 = 0.486
  样本 1: 所有位置都是类别 0 (真实)
          → 模型预测概率: 0.731 + 0.300 + 0.250 + 0.300 = 1.581
  总交集: 0.486 + 1.581 = 2.067

对于类别 1:
  样本 0: 位置 0 是类别 1 (真实)
          → 模型预测概率: 0.668
  样本 1: 没有位置是类别 1 (真实)
          → 模型预测概率: 0.000
  总交集: 0.668 + 0.000 = 0.668
```

## 完整代码示例

```python
import torch
import torch.nn.functional as F

# 准备数据
inputs = torch.tensor([
    [
        [0.100, 0.225, 0.261, 0.078],
        [0.668, 0.303, 0.214, 0.071],
        [0.122, 0.248, 0.236, 0.071],
        [0.110, 0.225, 0.289, 0.780]
    ],
    [
        [0.731, 0.300, 0.250, 0.300],
        [0.182, 0.300, 0.250, 0.300],
        [0.045, 0.300, 0.250, 0.200],
        [0.042, 0.100, 0.250, 0.200]
    ]
], dtype=torch.float32)

targets = torch.tensor([
    [1, 0, 0, 3],
    [0, 0, 0, 0]
], dtype=torch.long)

# 转换为 one-hot
num_classes = 4
targets_one_hot = F.one_hot(targets, num_classes).permute(0, 2, 1).float()

print("inputs shape:", inputs.shape)
print("targets_one_hot shape:", targets_one_hot.shape)

# 计算交集
elementwise_product = inputs * targets_one_hot
print("\n元素级乘积 (样本 0, 类别 0):")
print(elementwise_product[0, 0, :])

print("\n元素级乘积 (样本 0, 类别 1):")
print(elementwise_product[0, 1, :])

# 求和
intersection = torch.sum(elementwise_product, dim=(0, 2))
print("\n最终交集:")
print(intersection)
print(f"shape: {intersection.shape}")

# 手动验证类别 0
print("\n手动验证类别 0:")
sample0_class0 = elementwise_product[0, 0, :].sum()
sample1_class0 = elementwise_product[1, 0, :].sum()
print(f"样本 0, 类别 0 的和: {sample0_class0.item():.3f}")
print(f"样本 1, 类别 0 的和: {sample1_class0.item():.3f}")
print(f"总交集: {(sample0_class0 + sample1_class0).item():.3f}")
print(f"与 intersection[0] 比较: {intersection[0].item():.3f}")
```

## 关键理解

### 1. 元素级乘法的含义

```
inputs * targets_one_hot
```

- 当 `targets_one_hot[i, j, k] = 1.0` 时，保留 `inputs[i, j, k]` 的值
- 当 `targets_one_hot[i, j, k] = 0.0` 时，结果为 0.0
- **只保留真实标签位置的预测概率**

### 2. 求和维度的含义

```
dim=(0, 2)
```

- `dim=0`: 跨所有样本求和（batch 维度）
- `dim=2`: 跨所有位置求和（序列长度维度）
- **结果**: 每个类别的总交集值

### 3. 交集的物理意义

- **高交集值**: 模型在真实标签位置上的预测概率高，预测准确
- **低交集值**: 模型在真实标签位置上的预测概率低，预测不准确
- **零交集值**: 模型完全没有预测到该类别，或该类别在真实标签中不存在

## 总结

| 步骤 | 操作 | 形状变化 | 说明 |
|------|------|----------|------|
| 输入1 | `inputs` | `(2, 4, 4)` | 模型预测概率 |
| 输入2 | `targets_one_hot` | `(2, 4, 4)` | 真实标签 one-hot |
| 步骤1 | `inputs * targets_one_hot` | `(2, 4, 4)` | 元素级乘法 |
| 步骤2 | `torch.sum(..., dim=(0, 2))` | `(4,)` | 在 batch 和位置维度上求和 |
| 输出 | `intersection` | `(4,)` | 每个类别的总交集值 |

**关键公式**:
```
intersection[c] = Σᵢ Σₖ (inputs[i, c, k] * targets_one_hot[i, c, k])
```
其中：
- `i` 遍历所有样本 (batch)
- `k` 遍历所有位置 (sequence)
- `c` 是类别索引
